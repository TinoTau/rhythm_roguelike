# 🧩 流派自我确认 UI 设计规范（V1.0）

## 设计目标
在**不打断战斗、不教学的前提下**，
让玩家在事后确认：
> “原来我刚才是在这样表达。”

---

## 核心原则
- 只在结算出现
- 不作为操作提示
- 不评价对错，只描述结果

---

## UI 结构

### 1. 表达总结条
- 文案示例：
  - “节奏密集，爆发提前”
  - “结构稳定，段数偏多”
  - “高风险表达，效率极高”

---

### 2. 听感关键词（2–3 个）
- 示例：
  - 紧凑 / 松弛 / 激进 / 稳定
  - 压迫 / 空间感 / 断裂

---

### 3. 可选流派标签（弱提示）
- 示例：
  - "偏向：节奏表达型"（根据统一设计：战斗不是对抗敌人，而是表达主导）
  - "偏向：空间铺陈型"

> 标签永远在最后出现，且可忽略。

---

## 禁止事项
- 禁止显示“推荐流派”
- 禁止高亮“正确答案”
- 禁止影响下一局选择

---

# Boss 行为脚本设计（人格化流派）

## 设计立场（根据统一设计文档）

> **Boss 是强势表达者，而非攻击者。**  
> Boss 被设计为：单一、极其稳定的强势流派，不断重复、覆盖、模仿玩家。

玩家的挑战不是打倒 Boss，而是：**在其强大表达下仍不动摇**。

Boss 的崩解，意味着：**合奏主导权的转移**。

---

## 通用阶段

### Phase 1 宣告
- 稳定重复自身动机
- 干扰：空间占用（其他流派的演奏行为）

### Phase 2 对位
- 模仿并歪曲玩家表达
- 干扰：情绪反射（被误解）

### Phase 3 压制
- 加强节奏与结构压力
- 干扰：节奏压迫 + 段落撕裂（风格稀释、被同化）

### Phase 4 主导权转移
- 若玩家表达持续清晰
  → Boss 主导权崩塌（合奏主导权转移）
- 否则 Boss 完全同化合奏（玩家表达被淹没）

---

## Boss 特殊机制（根据 battle.txt 补充）

### 污染卡池增强

**机制**：
- 直接污染玩家的卡池（每条音轨增加 1/3 的污染牌）
- 考验玩家是否真的理解自己正在构筑的流派

**战斗状态**：
- 玩家一般都已经有了成熟的流派和足够的道具
- 需要在污染增强的情况下识别哪些是自己的牌，哪些是污染牌
- 即使没抽到想要的牌，打不出想要的效果，也要通过增加段落来完成战斗

**设计目的**：
- Boss 行为只是为了增加戏剧性
- 让玩家在识别出哪些是自己的牌，哪些是污染牌的情况下，也没抽到想要的牌，打不出想要的效果，迫不得已增加段落

---

## 胜利条件

**Boss 不再主导表达** = 玩家的表达成为合奏中的主导声音

---

## 干扰系统的统一作用（补充）

根据统一设计文档，干扰不限制操作，而是污染表达：

- **节奏漂移** → 说话被打断
- **动机腐蚀** → 重复变得疲惫
- **情绪反射** → 被误解
- **风格稀释** → 被同化

这些同时是：
- 世界在发声
- 战斗在进行
- 情绪压力的来源

---

## 与统一设计理念的对应

| 统一设计层面 | Boss 行为体现 |
|------------|------------|
| 世界观 | Boss 是其他流派/系统化演奏者/强势表达的具象化 |
| 战斗系统 | 干扰 = 其他流派的演奏行为，胜利 = 成为合奏中的主导声音 |
| 情绪体验 | 在压力下保持自我，表达被识别 |
